AutoRentNexus: Vehicle Rental Management System

AutoRent Nexus is a Java-based system designed to automate vehicle management, bookings, and customer profiles for car rental agencies. The application enhances rental efficiency by providing real-time vehicle availability tracking, cost calculations, and streamlined booking processes. By automating core functions such as vehicle management, customer service, and rental transactions, AutoRent Nexus aims to reduce manual workloads and improve overall customer satisfaction in the car rental industry.

Tools Used: Java, JUnit, Maven, Lucidchart, GitHub, Visual Studio Code

Development Process & Methodologies

Agile Methodology

The development of AutoRent Nexus followed Agile principles, allowing for iterative development and continuous improvement. Regular feedback was incorporated to refine features such as booking confirmation, vehicle availability tracking, and invoice generation.

Testing

JUnit was integrated for unit testing critical components of the system, ensuring reliability and correctness of functionalities such as booking calculations, vehicle availability checks, and customer profile management.

Maven Integration

Maven was used for project management and build automation, simplifying dependency management and facilitating a consistent build process across the development environment.

User Stories

Class Diagram

The class diagram for AutoRent Nexus outlines the key classes and their relationships within the system. The Admin class oversees vehicle management, bookings, and customer profiles. Customer stores personal details and rental history, while Booking manages booking information and cost calculations. The Vehicle class captures details like model, rental price, and availability, linked to VehicleType, which defines vehicle categories and daily rates. Invoice handles billing for each booking, and Rents maintains a record of all rented vehicles. These interconnected classes form the backbone of AutoRentNexus, supporting streamlined rental management.

System Capabilities and Key User Actions

Admin Account Creation and Login

Admins can create an account and log in to the system. After logging in, they have access to functionalities such as managing vehicles and customers, and processing bookings.

Vehicle Management

  • Add Vehicles: Input vehicle details including model, type, and rental price.
  • Update Vehicles: Modify existing vehicle information.
  • Remove Vehicles: Delete vehicles from the system.
  • View Vehicles: Display a list of all vehicles with their current status.

Customer Management

  • Create Customer Profiles: Enter customer details like name, email, phone, and address.
  • Edit Customer Profiles: Update customer information.
  • Remove Customer Profiles: Delete customers from the system.
  • View Customers: Display a list of all customers.

Booking Process

  • Search Available Vehicles: Based on date range and vehicle type.
  • Confirm Bookings: Reserve vehicles for customers, ensuring no double bookings occur.
  • Calculate Booking Costs: Total cost is computed based on rental period and vehicle type.
  • Generate Invoices: Create invoices for confirmed bookings.

Email Notification

Upon confirming a booking, the system generates an invoice and simulates sending an email to the customer with the booking confirmation and invoice details.

Code Highlights

Implementing Real-Time Vehicle Availability Tracking

Code Snippet: Vehicle.java

Challenge: Ensuring that vehicle availability is accurately tracked to prevent double bookings and scheduling conflicts.

Solution:

  • Booking List Maintenance: Each Vehicle object maintains a list of its associated bookings.
  • Availability Check Method: Implemented isAvailableForDates(Date startDate, Date endDate) to check availability against existing bookings.
  • State Update on Booking Confirmation: Vehicle availability status is updated when a booking is confirmed.

Reflection: This method provides a reliable way to track vehicle availability, ensuring that the booking process is robust and prevents conflicts. It underscores the importance of careful state management in object-oriented programming.

Calculating Booking Costs Based on Vehicle Type and Rental Period

Code Snippet: Booking.java

Challenge: Accurately calculating the total cost of a booking by considering the vehicle type's daily rate and the rental duration.

Solution:

  • Cost Calculation Method: Implemented calculateCost(double dailyRate, int numberOfDays) to compute the total cost.
  • Vehicle Type Integration: Utilized VehicleType enum to get the daily rate for the specific vehicle type.
  • Duration Calculation: Calculated the number of days between the start and end dates.

Reflection: This functionality ensures transparent and accurate billing, enhancing customer trust. It highlights the critical role of precise calculations in financial aspects of software applications.

Integrating JUnit for Automated Testing

Code Snippet: VehicleTest.java

Challenge: Ensuring the reliability of critical methods through automated unit testing.

Solution:

  • Test Cases for Availability: Wrote JUnit tests for isAvailableForDates to verify correct behavior under various scenarios.
  • Boundary Testing: Included tests for overlapping and non-overlapping date ranges.
  • Setup and Teardown: Utilized JUnit annotations to manage test setup.

Reflection: Integrating JUnit testing improved code reliability and facilitated early detection of bugs. It reinforced best practices in test-driven development.
